make(1) 本地用户命令 make(1)

姓名
       make - GNU make 工具，用于维护程序组

说明
       make [ -f makefile ] [ options ] ...[ 目标 ] ...

警告
       本手册是 GNU make 文档的摘录。  它是

说明
       make 实用程序的目的是自动确定哪个
       需要重新编译大型程序的各个部分，并发出以下命令
       重新编译。  该手册介绍了
       由 Richard Stallman 和 Roland McGrath 编写，是
       目前由保罗-史密斯维护。  我们的示例显示的是 C 程序、
       因为它们是最常见的，但您也可以在任何编程中使用 make
       语言，其编译器可通过 shell 命令运行。  事实上，make
       并不局限于程序。  您可以用它来描述任何任务，其中
       只要在运行时，某些文件就必须从其他文件中自动更新。
       其他变化。

       要准备使用 make，你必须编写一个名为 makefile 的文件，其中包括
       描述了程序中文件之间的关系，而状态
       更新每个文件的命令。  在程序中，通常
       可执行文件是由目标文件更新的，而目标文件又是由
       编译源文件。

       一旦有了合适的 makefile，每次更改源代码时
       文件的简单 shell 命令：

		make

       就足以执行所有必要的重新编译。  make 程序
       使用 makefile 数据库和
       文件，以决定哪些文件需要更新。  对于每个
       在这些文件中，它会发出数据库中记录的命令。

       make 执行 makefile 中的命令，以更新一个或多个目标
       名称，其中名称通常是一个程序。  如果没有 -f 选项、
       make 将查找 makefile GNUmakefile、makefile 和 Makefile、
       按顺序排列。

       通常情况下，您应该将 makefile 称为 makefile 或 Makefile。
       (我们推荐 Makefile，因为它出现在靠近
       在目录列表的开头，靠近其他重要文件，如
       如 README）。第一个被选中的名称 GNUmakefile 不推荐用于
       大多数 makefile 都使用这个名称。  如果您的 makefile 是
       是 GNU make 特有的，其他版本将无法理解
       的内容。  如果 makefile 为"-"，则读取标准输入。

       如果一个目标依赖于先决条件文件，而这些先决条件文件有
       被修改，或者如果目标没有
       不存在。选项
       -b、-m
            为了与其他版本的
            使。

       -B，-always-make
            无条件实现所有目标。

       -C dir, --directory=dir
            在读取 makefile 或执行
            其他任何选项。  如果指定了多个 -C 选项，则每个选项都是
            相对于前一个解释：-C / -C 等是
            相当于 -C /etc。  这通常与递归
            的调用。

       -除正常处理外，打印调试信息。  打印
            调试信息显示，哪些文件正在被考虑用于
            重新制作，比较哪些文件时间，以及与哪些文件时间比较
            结果，哪些文件实际上需要重新制作，哪些隐含的
            考虑和应用哪些规则--一切
            有趣的是，"制造 "是如何决定做什么的？

       --调试[=FLAGS]
            除正常处理外，打印调试信息。  如果
            如果省略 FLAGS，则行为与 -d 相同。
            指定。  FLAGS 可以是一个，用于所有调试输出（与使用
            -d），b 表示基本调试，v 表示更详细的基本调试，i
            用于显示隐式规则， j 用于详细说明调用
            命令，而 m 则用于在重制 makefile 时进行调试。

       -e，--环境覆盖
            环境变量优先于
            变量。

       +-f file, --file=file, --makefile=FILE
            将文件用作 makefile。

       -i，--忽略错误
            忽略重制文件命令中的所有错误。

       -I dir, --include-dir=dir
            指定要搜索包含的 makefile 的目录 dir。  如果
            多个 -I 选项用于指定多个目录，而
            目录的搜索顺序。  与
            的参数，用 -I 标志给出的目录
            可以直接跟在标志后面：-Idir 和 -I
            dir.  允许使用这种语法是为了与 C
            预处理器的 -I 标志。

       -j[工作]，--工作[=工作］
            指定同时运行的任务（命令）数量。  如果
            如果有多个 -j 选项，则最后一个有效。  如果
            选项时，make 将不会限制
            可同时运行的工作数量。

       -k，--继续
            出错后尽可能继续。  当目标
            失败者和依赖于失败者的人是无法重塑的。
            这些目标的其他依赖关系也可以同样处理。

       -l[负载]，--负载平均值[=负载］
            指定在出现以下情况时不启动新工作（命令
            有其他工作在运行，且平均负载至少为负载（a
            浮点数）。  无参数时，删除之前的加载值
            限制。

       -L、--检查系统连接时间
            使用符号链接和目标之间的最新 mtime。

       -n、--just-print、--dry-run、--recon
            打印将执行但未执行的命令
            他们

       -o file, --old-file=file, --assume-old=file
            不要重制文件，即使该文件比它的
            依赖关系，并且不会因
            文件。  从本质上讲，该文件被视为非常老的文件，其规则
            将被忽略。

       -p，--打印数据库
            打印由以下结果生成的数据库（规则和变量值
            读取 makefile；然后像往常一样执行，或像其他情况一样执行
            指定的版本。  这也会打印由
            -v开关（见下文）。  要打印数据库而不尝试
            重新制作任何文件，请使用 make -p -f/dev/null 命令。

       -q，--问题
            问题模式  不要运行任何命令，也不要打印任何内容；
            只是返回一个退出状态，如果指定的目标
            为最新，否则为非零。

       -r,--不建立规则
            取消使用内置隐含规则。  同时清除
            后缀规则的默认后缀列表。

       -R，--无内置变量
            不要定义任何内置变量。

       -s、--无声、--安静
            静音操作；执行命令时不打印。

       --S、--不继续、--停止
            取消 -k 选项的效果。  没有必要
            除了在递归 make 中，-k 可能会继承自
            通过 MAKEFLAGS 或在 MAKEFLAGS 中设置 -k 时，在您的
            环境

       -t，-触摸
            触摸文件（将其标记为最新，但不会进行真正的更改）
            而不是运行它们的命令。  这是用来假装
            命令，以便在今后调用
            使。

       -v,--版本
            打印制作程序的版本，加上版权和
            作者，并注明不提供担保。

       -w，--打印目录
            打印包含前后工作目录的信息
            其他处理。  这可能有助于追踪错误
            从复杂的递归 make 命令嵌套中解脱出来。

       --无打印目录
            关闭 -w，即使它已隐式开启。

       -W文件, --what-if=file, --new-file=file, --assume-new=file
            假装目标文件刚刚被修改。  使用时
            使用 -n 标志时，将显示如果您

            在运行 make 之前对给定文件执行 touch 命令。
